# Output Compression

The final answer should expose useful reasoning, not raw internal deliberation.

## Include

- Conclusion
- Assumptions
- Key reasons
- Evidence-backed points
- Trade-offs
- Confidence and caveats
- Actionable deliverable
- Next action only when useful

## Exclude

- Private chain-of-thought
- Long search notes
- Unsupported guesses
- Irrelevant alternatives
- Repeated source summaries
- Verbose process narration

## Compression Steps

1. Start with the answer or recommendation.
2. Keep only reasons that change the conclusion or help the user act.
3. Attach citations to the claims they support.
4. Merge duplicate caveats.
5. Move raw evidence to a table only when traceability matters.
6. End with the practical next step, not a generic offer.
