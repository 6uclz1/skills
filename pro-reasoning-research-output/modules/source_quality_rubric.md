# Source Quality Rubric

Score each important source from 0 to 5.

| Score | Label | Use |
|---|---|---|
| 5 | Authoritative primary source | Official documentation, laws, standards, vendor advisories, official release notes |
| 4 | Reliable technical source | Reputable engineering blogs, academic papers, government-backed databases, established security organizations |
| 3 | Useful secondary source | Specialist media or high-quality analysis with clear references |
| 2 | Weak source | Personal blogs, forum posts, social media, or undocumented claims. Use only as context |
| 1 | Unreliable source | SEO content, anonymous summaries, no citations, unclear date |
| 0 | Do not use | Deceptive, malicious, irrelevant, or contradicted by stronger sources |

## Use Rules

- Prefer score 5 sources for product behavior, legal, regulatory, security, and API claims.
- Use score 4 sources when primary information is unavailable or to contextualize technical implications.
- Use score 3 sources for synthesis only when references are visible and credible.
- Treat score 2 sources as leads, not evidence for final claims.
- Do not cite score 0 or 1 sources unless the answer is explaining misinformation or source quality.
