# Code Reasoning Examples

## Debugging

Use Audit Mode when the user asks why a system fails.

- Define expected behavior.
- Identify the observed failure.
- Read relevant code and tests.
- Form a minimal hypothesis.
- Add or run a failing test when feasible.
- Implement the smallest fix.
- Verify and refactor.

## Design

Use Design Mode for architecture or implementation plans.

- Define the contract first.
- Keep state and logic separated.
- Prefer existing local patterns.
- Add abstractions only when they reduce real complexity.
- Tie recommendations to tests or acceptance criteria.
