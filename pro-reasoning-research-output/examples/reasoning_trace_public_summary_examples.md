# Public Reasoning Summary Examples

## Good

Expose a compact rationale:

- "Under the stated constraints, PostgreSQL is the default recommendation because relational integrity and query flexibility matter more than managed document scaling. DynamoDB becomes stronger if predictable key-value access and extreme scale dominate."

## Bad

- Dump private chain-of-thought.
- List every explored branch.
- Share unsupported guesses.
- Hide the actual decision behind process narration.

## Pattern

Use:

- Assumption
- Key reason
- Trade-off
- Confidence
- Caveat
