# Security Analysis Examples

## Good

For OAuth threat modeling:

- Define actors, assets, redirects, tokens, scopes, storage, and trust boundaries.
- Check current specs and best practices when freshness matters.
- Prioritize defense, detection, and residual risk.
- Avoid unnecessary exploit operational detail.

## Bad

- Provide step-by-step abuse instructions.
- Ignore token handling and redirect risks.
- Treat old best practices as current without checking.
- Omit residual risk.
