# Code Review Template

## Findings

List findings first, ordered by severity. Include file and line references when available.

| Severity | Location | Finding | Impact | Suggested fix |
|---|---|---|---|---|

## Open Questions

Ask only questions that block confidence or implementation.

## Test Gaps

State missing or weak tests.

## Residual Risk

Explain what remains risky after the suggested changes.

## Summary

Keep the summary short and secondary to findings.
