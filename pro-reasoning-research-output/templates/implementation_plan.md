# Implementation Plan Template

## Goal

Define the outcome and acceptance criteria.

## Scope

State in scope, out of scope, dependencies, and constraints.

## Architecture Or Workflow

Describe components, interfaces, data flow, and ownership.

## Phases

1. Exploration
2. Red
3. Green
4. Refactoring
5. Verification
6. Release or handoff

## Tests And Validation

Map requirements to tests, checks, or review gates.

## Risks

List technical, operational, security, and timeline risks.

## Deliverables

Name the files, artifacts, or decisions to produce.
