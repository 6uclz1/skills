# v0.2.0 Release Notes

Date: 2026-05-03

## Summary

This release explicitly includes `compose-music` as an implementation-oriented Codex skill. The skill now has a schema-first handoff contract, deterministic MIDI conversion helpers, validation, an eval harness, expanded genre playbooks, and release documentation.

## Included compose-music Assets

- `compose-music/SKILL.md`
- `compose-music/agents/openai.yaml`
- `compose-music/references/`
- `compose-music/scripts/`
- `compose-music/tests/`
- `evals/compose-music.prompts.csv`
- `evals/compose-music.checks.json`
- `evals/run-compose-music-evals.mjs`

## User-Facing Changes

- Shorter invocation metadata for better skill selection.
- `composition_spec` v1 JSON Schema.
- Validation for fake browser paths, section totals, note ranges, timing metadata, role references, and identity carriers.
- Handoff plan generation before `$ableton-cli` operation.
- 20 genre playbooks with concrete avoid lists.

python3 -m unittest compose-music/tests/test_composition_scripts.py
