# Sound Design Palettes

Use this reference for Ableton-ready sound direction without hard-coding browser item paths, kits, presets, or device targets. Provide search queries and musical roles; let `$ableton-cli` or the operator resolve exact browser results.

## Drums

- Search query examples: `Drum Rack`, `Kit`, `dry electronic kit`, `909 kit`, `breakbeat kit`
- Role: groove identity, transient structure, section contrast
- Parameters to consider after loading: kick decay, hat brightness, clap/snare length, swing, velocity range
- Handoff note: exact rack or kit must come from active browser search results

## Bass

- Search query examples: `Operator bass`, `Drift bass`, `Analog bass`, `Wavetable bass`, `808 bass`
- Role: low-end identity, root clarity, kick relationship
- Parameters to consider after loading: filter cutoff, envelope decay, saturation, mono width
- Handoff note: describe whether the bass avoids, doubles, or answers kick hits

## Chords

- Search query examples: `Wavetable pad`, `Analog keys`, `Electric keys`, `Chord stab`, `Juno pad`
- Role: harmonic color, midground motion, emotional frame
- Parameters to consider after loading: filter cutoff, attack, release, voice count, chorus width
- Handoff note: keep low chord tones sparse; bass owns the root weight

## Lead

- Search query examples: `Wavetable lead`, `Analog lead`, `Pluck`, `Mono lead`, `Vocal chop`
- Role: foreground motif, call-and-response, peak identity
- Parameters to consider after loading: portamento, filter envelope, delay send, octave, modulation
- Handoff note: keep the motif short enough to vary by rhythm or ending note

## Texture/FX

- Search query examples: `Noise`, `Atmosphere`, `Field recording`, `Riser`, `Impact`, `Reverb`
- Role: transitions, background motion, width, section markers
- Parameters to consider after loading: filter, volume automation, reverb time, sidechain movement
- Handoff note: texture should support arrangement clarity, not hide weak writing
